À propos de ce cours

Tu sais déjà coder en Python. Tu sais peut-être déjà te débrouiller dans Excel. Mais entre les deux, il y a un trou noir : VBA (Visual Basic for Applications), le langage qui vit à l'intérieur d'Excel et qui automatise tout ce que tu fais à la souris. Nettoyer un export sale en 2 secondes, générer un rapport mensuel en un clic, parcourir 12 feuilles sans copier-coller : c'est ça, VBA.

Beaucoup de Data Analysts l'évitent parce qu'il a une réputation de « vieux truc des comptables ». Erreur. Sur le terrain, VBA est partout dans les entreprises : des milliers de classeurs métiers tournent grâce à lui. Savoir le lire, le corriger et l'écrire, c'est une compétence qui te rend immédiatement utile — et qui te démarque dans un entretien.

La promesse de ce cours : on commence sans écrire une ligne de code (l'enregistreur de macros fait le travail), puis on apprend à lire ce que l'enregistreur produit, et enfin on code nous-mêmes, proprement, jusqu'à automatiser un tableau de bord complet. Du clic au code pro, sans sauter d'étape.

🚀
Astuce de pro · pourquoi apprendre VBA quand on connaît déjà Python ?

Parce que tu ne contrôles pas toujours l'environnement du client. Sur un poste verrouillé sans droits d'installation, sans Python, sans pip, il te reste… Excel. Et Excel embarque VBA gratuitement, sans rien installer. Python pour l'analyse lourde, VBA pour automatiser le classeur que le métier utilise tous les jours : les deux ne s'opposent pas, ils se complètent.

VBA en une phrase

VBA (Visual Basic for Applications) est le langage de programmation intégré à Excel — et à tout Office. Il te permet de piloter Excel par du code : lire et écrire des cellules, mettre en forme, parcourir des milliers de lignes, créer des graphiques, ouvrir d'autres classeurs… tout ce que tu fais à la souris, VBA peut le refaire, en boucle, en une fraction de seconde.

💡
Le savais-tu ? · Une « macro » n'est rien d'autre qu'un programme

Le mot macro fait peur, mais une macro est simplement une procédure VBA — un bloc de code nommé que tu peux rejouer. Quand tu « enregistres une macro », Excel écrit ce code pour toi. Tu fais déjà du code sans le savoir.

Pourquoi toi, qui fais du Python ?

Tu pourrais te dire : « j'ai pandas, pourquoi VBA ? ». Trois raisons très concrètes :

  • VBA vit dans Excel. Pas d'export CSV, pas d'environnement à installer, pas de fichier à rouvrir ailleurs. Le traitement s'exécute là où sont les données — souvent ce que veut le client.
  • L'utilisateur final n'a rien à installer. Tu livres un .xlsm avec un bouton. La personne clique. C'est tout. Aucun Python sur sa machine.
  • C'est une niche rentable. Beaucoup d'entreprises tournent sur des classeurs critiques. Peu de gens savent les maintenir. Rareté = valeur.
🚀
Astuce de pro · Le bon outil au bon endroit

Gros volumes, machine learning, pipelines ? Python. Automatiser un reporting Excel que dix collègues utilisent chaque lundi ? VBA. Un bon Data Analyst ne choisit pas un camp — il choisit l'outil adapté au contexte.

✅ Tu sais maintenant

  • Définir VBA : le langage qui pilote Excel par du code.
  • Qu'une « macro » est juste une procédure que tu peux rejouer.
  • Citer 3 raisons d'apprendre VBA même en venant de Python.

Préparer le terrain : onglet Développeur, sécurité, fichier .xlsm

En bref

Tu vas activer l'onglet Développeur, comprendre pourquoi Excel bloque les macros par défaut, et enregistrer ton classeur au bon format pour qu'il n'efface pas ton code.

🎬Vidéo — Activation de l'onglet Développeur, pas à pas (à insérer).

Faire apparaître l'onglet Développeur

C'est ton tableau de bord VBA. Il est masqué par défaut. Pour l'afficher :

Fichier → Options → Personnaliser le ruban → coche Développeur dans la colonne de droite → OK.

Un nouvel onglet Développeur apparaît dans le ruban. Tu y trouveras les boutons Visual Basic, Macros, Enregistrer une macro.

⚠️
Attention · La sécurité des macros

Une macro peut tout faire sur ton ordinateur — y compris des choses malveillantes. C'est pourquoi Excel bloque les macros par défaut dans les fichiers téléchargés. Quand tu ouvres un .xlsm venu d'ailleurs, une bande jaune « Avertissement de sécurité » s'affiche.

Pour tes propres fichiers, clique sur Activer le contenu. Pour un fichier inconnu, n'active jamais sans savoir ce qu'il contient : ouvre d'abord l'éditeur VBA et lis le code.

Le format qui ne perd pas ton code

Un .xlsx classique n'enregistre pas les macros : si tu codes puis sauvegardes en .xlsx, Excel t'avertit et supprime tout ton VBA. Il faut le format classeur Excel prenant en charge les macros :

Fichier → Enregistrer sous → type Classeur Excel (prenant en charge les macros) (*.xlsm).

💡
Le savais-tu ? · Le « m » de .xlsm

Le m signifie macro-enabled. Tu rencontreras aussi .xlsb (binaire, plus léger et rapide pour les gros fichiers) et .xlam (un complément : une boîte à outils de macros réutilisable dans tous tes classeurs).

✅ Tu sais maintenant

  • Afficher l'onglet Développeur.
  • Pourquoi Excel bloque les macros, et comment activer le contenu en sécurité.
  • Enregistrer en .xlsm pour conserver ton code.

  4. Onglet Développeur → Arrêter l'enregistrement.

1.4

Exécuter une macro : raccourci, bouton, et pas-à-pas (F8)

En bref

Tu vas relancer ta macro de 4 façons différentes — dont le mode pas-à-pas avec F8, l'outil qui te montre le code s'exécuter ligne par ligne.

🎬Vidéo — Les 4 façons de lancer une macro (à insérer).

Quatre déclencheurs

  • La boîte Macros : Développeur → Macros (ou Alt+F8) → choisis ta macro → Exécuter.
  • Un raccourci clavier : celui que tu as défini à l'enregistrement (ex. Ctrl+Maj+M).
  • Un bouton sur la feuille : Développeur → Insérer → bouton de formulaire → tu le dessines, puis tu lui associes la macro. Idéal pour livrer à un utilisateur non technique.
  • Depuis l'éditeur : place le curseur dans la macro et appuie sur F5.

Le pas-à-pas : voir le code vivre

C'est l'outil d'apprentissage le plus puissant de VBA. Dans l'éditeur, place le curseur dans ta macro et appuie sur F8 de façon répétée : chaque pression exécute une seule ligne. Mets l'éditeur et Excel côte à côte : tu vois la cellule changer à l'instant où la ligne s'exécute. La ligne sur le point d'être exécutée est surlignée en jaune.

💡
Le savais-tu ? · F8, le ralenti de l'analyste

Quand une macro fait « quelque chose de bizarre », le réflexe pro n'est pas de relire le code en silence : c'est de le passer en F8 et de regarder où ça dérape. On verra le débogage en profondeur en saison 4.

✅ Tu sais maintenant

  • Lancer une macro de 4 manières (boîte, raccourci, bouton, F5).
  • Associer une macro à un bouton pour un utilisateur final.
  • Exécuter en pas-à-pas avec F8 et observer chaque ligne.
1.5

Le piège classique : références absolues vs relatives

En bref

Tu vas comprendre la case que tout débutant oublie de cocher : « Utiliser les références relatives ». Selon son état, la même macro se comporte radicalement différemment.

🎨Manga — Synapto lance la même macro depuis deux cellules : deux résultats opposés, surprise garantie (à insérer).

Deux façons d'enregistrer un déplacement

Quand tu enregistres, le bouton « Utiliser les références relatives » (onglet Développeur) change tout :

  • Absolu (par défaut) : l'enregistreur note des adresses fixes. « Va en A1 » visera toujours A1, peu importe où tu te trouves.
  • Relatif (bouton activé) : l'enregistreur note des déplacements. « Descends d'une ligne » partira de la cellule active, où qu'elle soit.

2.1

L'éditeur VBE, les modules et l'anatomie d'une macro

En bref

Tu vas ouvrir l'éditeur VBA (VBE), comprendre où vit ton code, créer un module, et disséquer une macro en ses parties (Sub, instructions, End Sub).

🎬Vidéo — Visite guidée de l'éditeur VBE (à insérer).

Entrer dans le VBE

Depuis Excel, appuie sur Alt+F11 (ou Développeur → Visual Basic). Une fenêtre séparée s'ouvre : c'est le VBE (Visual Basic Editor). À gauche, l'Explorateur de projets liste ton classeur, ses feuilles, et tes modules.

Où vit le code ?

VBA range le code à différents endroits :

  • Les modules standard (Module1, Module2…) : pour tes macros générales. C'est là que tu écriras le plus souvent.
  • Les modules de feuille et ThisWorkbook : pour le code lié à des événements (à l'ouverture du classeur, au changement d'une cellule…). On y reviendra plus tard.

Pour créer un module : dans le VBE, menu Insertion → Module. Un Module1 apparaît, vide, prêt à recevoir ton code.

L'anatomie d'une macro

Sub annonce une procédure (un bloc d'actions). Entre Sub et End Sub, chaque ligne est une instruction. Le nom (DireBonjour) est celui qui apparaîtra dans la liste des macros.

🚀
Astuce de pro · Toujours commencer par Option Explicit

Tape Option Explicit tout en haut de chaque module. Cela oblige à déclarer chaque variable et t'évite des bugs idiots (une faute de frappe dans un nom de variable devient une erreur visible, pas un silence). Active-le automatiquement : VBE → Outils → Options → Exiger la déclaration des variables.

✅ Tu sais maintenant

  • Ouvrir le VBE avec Alt+F11 et lire l'Explorateur de projets.
  • Créer un module et savoir où vit le code.
  • Reconnaître Sub, instructions et End Sub, et activer Option Explicit.
2.2

Du code enregistré au code propre

En bref

Tu vas transformer le code « brut » de l'enregistreur en code lisible et efficace, en supprimant le motif Select/Selection qui le ralentit.

🎨Manga — Synapto taille dans un bloc de code touffu ; il reste 3 lignes nettes (à insérer).

Le défaut de l'enregistreur : il sélectionne tout

L'enregistreur reproduit tes gestes humains : tu cliques une cellule (sélection), puis tu agis. En code, ça donne un .Select suivi de Selection.…. Mais VBA n'a pas besoin de sélectionner pour agir : tu peux parler à une cellule directement.

La règle : chaque fois que tu vois X.Select puis Selection.Y, fusionne-les en X.Y. Plus court, plus clair, et nettement plus rapide sur de gros traitements.

💡
Le savais-tu ? · Pourquoi Select ralentit

Chaque .Select force Excel à redessiner l'écran et à déplacer réellement le curseur. Sur 10 000 lignes, c'est 10 000 rafraîchissements inutiles. En t'adressant aux cellules sans les sélectionner, tu évites tout ce travail. On chiffrera ce gain en saison 4.

✅ Tu sais maintenant

  • Repérer le motif Select/Selection dans le code généré.
  • Le réécrire en s'adressant directement à l'objet.
  • Pourquoi ce nettoyage rend le code plus rapide et plus lisible.
2.3

Dialoguer : MsgBox et InputBox

En bref

Tu vas afficher des messages à l'utilisateur avec MsgBox et lui demander une saisie avec InputBox — les deux briques de toute interaction simple.

🎬Vidéo — MsgBox d'information vs InputBox de saisie (à insérer).

MsgBox : afficher un message

Le 2e argument choisit l'icône (vbInformation, vbExclamation, vbCritical) ; le 3e est le titre de la fenêtre.

InputBox : demander une valeur

InputBox renvoie ce que l'utilisateur a tapé (toujours du texte). Tu le stockes dans une variable pour t'en servir ensuite.

⚠️
Attention · InputBox renvoie du texte, même pour un nombre

Si tu demandes un nombre et fais directement un calcul, gare aux surprises : "5" & "3" donne "53", pas 8. Convertis d'abord avec CDbl() ou Val(). On verra les conversions de type en saison 3.

🚀
Astuce de pro · MsgBox qui pose une question

MsgBox peut afficher des boutons Oui/Non et te dire ce que l'utilisateur a cliqué : If MsgBox("Continuer ?", vbYesNo) = vbYes Then …. Parfait pour confirmer une action irréversible (suppression de données, par exemple).

✅ Tu sais maintenant

  • Afficher un message avec icône et titre via MsgBox.
  • Demander et récupérer une saisie avec InputBox.
  • Te méfier du type texte renvoyé, et confirmer une action avec vbYesNo.

Tout est objet, et les objets sont emboîtés

VBA voit Excel comme des poupées russes. L'application contient des classeurs, chaque classeur contient des feuilles, chaque feuille contient des cellules. On descend la hiérarchie avec des points :

Range vs Cells : deux façons de viser une cellule

Range("B3") utilise l'adresse « humaine ». Cells(3, 2) utilise ligne puis colonne en nombres (ligne 3, colonne 2 = B3). Cells est idéal dans les boucles, où le numéro de ligne est une variable.

Les types qui couvrent 95 % des cas

  • Long : nombre entier (numéros de lignes, compteurs). Préfère-le à Integer, qui plafonne à 32 767.
  • Double : nombre décimal (montants, moyennes).
  • String : texte.
  • Boolean : True / False.
  • Variant : « n'importe quoi » (comme une variable Python). Pratique mais plus lent : à éviter quand tu connais le type.
🚀
Astuce de pro · Une déclaration ≠ une ligne par variable

Attention au piège : Dim i, j As Long ne type que j ; i devient un Variant. Pour typer les deux : Dim i As Long, j As Long. C'est une différence subtile mais classique en entretien.

✅ Tu sais maintenant

Tes propres fonctions (UDF) utilisables dans les cellules

En bref

Tu vas créer une UDF (User Defined Function) : une fonction maison que tu pourras taper dans une cellule comme =SOMME(), en utilisant Function … End Function.

🎬Vidéo — Écrire =PrixTTC(...) et l'utiliser dans la feuille (à insérer).

Sub vs Function

Une Sub fait des actions. Une Function renvoie une valeur — exactement comme une fonction Python avec return. En VBA, on « renvoie » en affectant le résultat au nom de la fonction :

Python
VBA

Une fois écrite dans un module, tape dans une cellule : =PrixTTC(100) → 120. Elle apparaît même dans l'auto-complétion d'Excel. Le _ en fin de ligne sert juste à couper une longue instruction sur deux lignes.

⚠️
Attention · Une UFD ne modifie pas la feuille

Une fonction appelée depuis une cellule ne peut que renvoyer une valeur dans sa propre cellule. Elle ne peut pas colorer d'autres cellules, en écrire ailleurs, ni supprimer des lignes. Pour ça, il te faut une Sub (lancée par un bouton). C'est une limite voulue par Excel, pas un bug.

🚀
Astuce de pro · Renvoyer une erreur propre

Si un calcul n'a pas de sens (division par zéro, par exemple), renvoie une vraie erreur Excel plutôt qu'un texte : MargePourcent = CVErr(xlErrDiv0) affiche #DIV/0! dans la cellule, cohérent avec les formules natives.

✅ Tu sais maintenant

  • Distinguer Sub (agit) et Function (renvoie).
  • Écrire une UDF avec arguments et Optional, et l'utiliser dans une cellule.
  • La limite d'une UDF et comment renvoyer une erreur Excel propre.
