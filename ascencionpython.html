Exercice 1 — Le premier cri du grimpeur

Affiche le message Bonjour le Mont Python ! à l'écran.

💡 Indice 1

La fonction qui affiche du texte dans la console s'appelle print().

💡 Indice 2

Le texte (une chaîne de caractères) doit être entouré de guillemets : doubles " ou simples '.

✅ Solution
print("Bonjour le Mont Python !")

Comment ça marche :

  • print() est une fonction intégrée à Python : elle affiche dans la console ce qu'on lui passe entre parenthèses. C'est ton premier outil pour voir le résultat de ton code.
  • Les guillemets disent à Python « ceci est du texte à afficher tel quel », et non une instruction ou un nom de variable.
Le savais-tu ?

Python ne tire pas son nom du serpent mais de la troupe comique britannique Monty Python, dont le créateur du langage, Guido van Rossum, était fan. C'est pour ça que la documentation officielle adore les clins d'œil à leurs sketchs — tu croiseras souvent spam et eggs dans les exemples au lieu de foo et bar.

Exercice 2 — Ton nom sur le carnet de cordée

Crée une variable nommée nom_grimpeur contenant "Alex", puis affiche : Bonjour, Alex ! Prêt pour l'ascension ?

💡 Indice 1

Pour créer une variable, on utilise le signe égal : nom_de_la_variable = valeur.

💡 Indice 2

Pour insérer la variable dans le message, sépare le texte et la variable par des virgules, ou utilise une f-string pour une insertion plus directe.

✅ Solution
nom_grimpeur = "Alex"
print("Bonjour,", nom_grimpeur, "! Prêt pour l'ascension ?")

# Version moderne, plus lisible, avec une f-string :
print(f"Bonjour, {nom_grimpeur} ! Prêt pour l'ascension ?")

Comment ça marche :

  • nom_grimpeur = "Alex" crée une « boîte » en mémoire, étiquetée nom_grimpeur, qui contient le texte "Alex".
  • Passé plusieurs arguments séparés par des virgules, print() les affiche à la suite en ajoutant un espace entre chacun.
  • Le f devant les guillemets active la f-string : tu insères une variable directement dans le texte en l'entourant d'accolades { }. C'est la méthode la plus lisible et la plus recommandée.
Astuce de pro — la f-string, ta meilleure amie

Depuis Python 3.6, la f-string est la façon la plus propre d'afficher des valeurs : f"Altitude : {altitude} m". En data, tu t'en serviras en permanence pour les messages de suivi (logs). Bonus pour déboguer (Python 3.8+) : écris print(f"{altitude=}") et Python affiche altitude=250 — le nom et la valeur, sans rien retaper. Un réflexe qui fait gagner un temps fou.

Exercice 3 — Un nombre dans le sac

Crée une variable altitude_actuelle et assigne-lui la valeur 100. Affiche ensuite : Nous sommes à 100 mètres d'altitude.

💡 Indice 1

Les nombres (entiers comme 100 ou décimaux comme 3.14) ne s'entourent jamais de guillemets.

💡 Indice 2

Affiche la variable numérique exactement comme du texte. Une f-string fera parfaitement l'affaire.

✅ Solution
altitude_actuelle = 100
print(f"Nous sommes à {altitude_actuelle} mètres d'altitude.")

Comment ça marche :

  • Sans guillemets, Python comprend que 100 est un nombre entier (type int) et non du texte.
  • La f-string convertit automatiquement le nombre en texte pour l'intégrer dans la phrase.
Attention — deux pièges du camp de base
  • Oublier les guillemets : print(Bonjour) déclenche une NameError, car Python cherche une variable nommée Bonjour qui n'existe pas.
  • Mal nommer une variable : un nom ne peut pas commencer par un chiffre ni contenir d'espace (2altitude et mon altitude sont interdits). La convention Python (PEP 8) : minuscules et underscores, comme altitude_max — c'est le style snake_case.

Exercice 4 — Le grimpeur progresse

Reprends altitude_actuelle. Initialise-la à 100 et affiche-la. Puis mets-la à jour à 250 et affiche la nouvelle valeur. Chaque affichage doit être explicite.

💡 Indice 1

Pour changer la valeur, réassigne simplement avec =. Tu ne recrées pas la variable, tu la mets à jour.

💡 Indice 2

Place un print() après chaque assignation pour voir l'évolution.

✅ Solution
altitude_actuelle = 100
print(f"Altitude initiale : {altitude_actuelle} mètres.")

altitude_actuelle = 250
print(f"Nouvelle altitude après progression : {altitude_actuelle} mètres.")

Comment ça marche :

  • La seconde assignation écrase l'ancienne valeur : 100 disparaît, remplacée par 250.
  • C'est la nature dynamique des variables en Python : leur contenu change au fil du programme.

Exercice 5 — Laisser des notes sur le chemin

Écris un programme qui affiche Python est fantastique !, précédé d'un commentaire qui explique ce que fait la ligne print.

💡 Indice 1

Un commentaire sur une ligne commence par un dièse #. Tout ce qui suit sur la même ligne est ignoré par Python.

💡 Indice 2

Place le commentaire juste au-dessus de la ligne print() pour la contextualiser.

Comment ça marche :

  • Le # marque le début d'un commentaire : Python l'ignore complètement à l'exécution.

Exercice 6 — Le relevé du camp

Un relevé regroupe plusieurs informations. Crée nom_camp = "Camp 1", altitude = 1200 et temperature = -3, puis affiche une seule ligne récapitulative : Camp 1 — altitude 1200 m, température -3 °C.

💡 Indice 1

Une variable par information. Un texte garde ses guillemets, les deux nombres n'en ont pas.

💡 Indice 2

Une f-string peut contenir plusieurs accolades : f"{a} ... {b} ... {c}".

✅ Solution
nom_camp = "Camp 1"
altitude = 1200
temperature = -3

print(f"{nom_camp} — altitude {altitude} m, température {temperature} °C.")

Comment ça marche :

  • Tu viens de créer un vrai relevé : plusieurs variables qui décrivent une même chose. C'est exactement l'idée d'une ligne de données en analyse — un camp ici, un client ou une transaction plus tard.
  • Une f-string unique assemble proprement les trois valeurs en une phrase lisible.

Exercice 7 — Le panneau du sentier

Affiche ce petit panneau sur trois lignes :

=== MONT PYTHON ===
Camp 1 -> Sommet
Bon courage !
💡 Indice 1

Chaque print() ajoute automatiquement un saut de ligne à la fin.

💡 Indice 2

Tu peux aussi insérer un saut de ligne manuel à l'intérieur d'une chaîne avec \n.

✅ Solution
# Méthode 1 : un print par ligne
print("=== MONT PYTHON ===")
print("Camp 1 -> Sommet")
print("Bon courage !")

# Méthode 2 : une seule chaîne avec des sauts de ligne \n
print("=== MONT PYTHON ===\nCamp 1 -> Sommet\nBon courage !")
  • print() termine toujours par un saut de ligne : trois print() = trois lignes.
  • \n est un caractère spécial qui force un saut de ligne à l'intérieur d'une chaîne. Pratique pour écrire plusieurs lignes en un seul appel.

Exercice 8 — L'apostrophe rebelle

Affiche exactement : J'adore l'Everest et ses "8 848 m". (avec l'apostrophe et les guillemets).

💡 Indice 1

Une apostrophe dans une chaîne entourée d'apostrophes pose problème. Entoure plutôt de guillemets doubles.

💡 Indice 2

Pour afficher un guillemet du même type que celui qui entoure la chaîne, échappe-le avec un antislash : \". Ou utilise des triples guillemets.

✅ Solution
# On entoure d'apostrophes... mais le texte en contient. On échappe les guillemets doubles :
print("J'adore l'Everest et ses \"8 848 m\".")

# Variante sans aucun échappement, avec des triples guillemets :
print('''J'adore l'Everest et ses "8 848 m".''')
  • L'antislash \" dit à Python « ce guillemet fait partie du texte, ce n'est pas la fin de la chaîne ».
  • Les triples guillemets ('''…''' ou """…""") peuvent contenir apostrophes et guillemets sans rien échapper — on les retrouvera pour les docstrings.

Exercice 9 — Première variable booléenne

Crée sommet_atteint = False et affiche Sommet atteint : False. Puis passe-la à True et réaffiche.

💡 Indice 1

True et False s'écrivent avec une majuscule et sans guillemets : ce ne sont pas des textes mais des booléens.

💡 Indice 2

Affiche la variable avec une f-string, comme un nombre.

✅ Solution
sommet_atteint = False
print(f"Sommet atteint : {sommet_atteint}")

sommet_atteint = True
print(f"Sommet atteint : {sommet_atteint}")
  • Le type bool n'a que deux valeurs : True et False (majuscule obligatoire).
  • Ces booléens sont le carburant des conditions qu'on verra au Camp 4 : « si le sommet est atteint, alors… ».

Exercice 10 — Carte d'identité du grimpeur

Crée prenom = "Junko", nom = "Tabei" et pays = "Japon". Affiche : Junko Tabei — Japon.

💡 Indice 1

Une f-string peut contenir plusieurs variables : f"{a} {b} — {c}".

💡 Indice 2

Pense à l'espace entre le prénom et le nom, et au tiret entouré d'espaces avant le pays.

  • On assemble trois variables dans une seule phrase lisible.
  • Petit clin d'œil : Junko Tabei fut la première femme à atteindre le sommet de l'Everest, en 1975. 🗻

Types & opérations

int, float, str, bool · opérateurs · casting · input() — manipuler et convertir

✓ Camp franchi

On sait stocker des informations : apprenons à les manipuler. Combiner du texte, faire des calculs, transformer une donnée d'un type à l'autre (le casting) et dialoguer avec l'utilisateur.

Exercice 1 — Le calcul de l'ascension

Tu es à 250 m et tu montes de 125 m. Crée altitude_depart et progression, calcule altitude_finale, puis affiche : Après progression, l'altitude est de 375 mètres.

💡 Indice 1

Utilise l'opérateur + pour additionner les deux nombres.

💡 Indice 2

La structure : nouvelle_variable = variable1 + variable2, puis une f-string pour l'affichage.

✅ Solution
altitude_depart = 250
progression = 125
altitude_finale = altitude_depart + progression

print(f"Après progression, l'altitude est de {altitude_finale} mètres.")

Comment ça marche :

  • Comme les deux variables sont des nombres, + fait une addition mathématique.
  • Stocker le résultat dans altitude_finale rend le code plus clair que de tout calculer dans le print().
  • Les autres opérateurs utiles : - (soustraction), * (multiplication), / (division), % (reste) et ** (puissance).

Exercice 2 — Le nom complet sur le permis

Crée prenom = "Marie" et nom = "Curie". Combine-les dans nom_complet et affiche le résultat.

💡 Indice 1

Avec des chaînes de caractères, + ne fait pas une addition mais une concaténation : il colle les chaînes bout à bout.

💡 Indice 2

N'oublie pas d'ajouter un espace (" ") entre le prénom et le nom.

✅ Solution
prenom = "Marie"
nom = "Curie"
nom_complet = prenom + " " + nom

print(nom_complet)

Comment ça marche :

  • Le même opérateur + se comporte différemment selon le type : il additionne des nombres mais concatène des chaînes. On appelle ça la surcharge d'opérateur.
  • prenom + " " + nom colle "Marie", un espace, puis "Curie" → "Marie Curie".
Le savais-tu ?

En Python, tout est objet — même les nombres. Plus surprenant : le type bool est en réalité un sous-type de int. Du coup True vaut 1 et False vaut 0, et True + True renvoie… 2 ! C'est étrange, mais redoutablement pratique en data : sum([True, False, True]) donne 2, c'est-à-dire le nombre de valeurs vraies. On compte ainsi en une ligne les lignes qui respectent une condition.

Exercice 3 — Connaître la nature d'une donnée

Affiche le type de chacune de ces valeurs avec la fonction type() : 42, 3.14, "Everest" et True.

💡 Indice 1

La fonction type(valeur) renvoie le type d'une valeur.

💡 Indice 2

Combine-la avec print() : print(type(42)).

✅ Solution
print(type(42))         # <class 'int'>   → entier
print(type(3.14))       # <class 'float'> → nombre à virgule
print(type("Everest"))  # <class 'str'>   → chaîne de caractères
print(type(True))       # <class 'bool'>  → booléen

Comment ça marche :

  • Les quatre types de base : int (entiers), float (décimaux), str (texte) et bool (True/False).
  • type() est un réflexe de data analyst : quand un calcul renvoie un résultat bizarre, la première question est souvent « de quel type est cette donnée ? ». Tu retrouveras cette idée avec le dtype des colonnes en Pandas.

Exercice 4 — Le piège du mélange des types

Crée message = "Mon âge est " et age = 30. Tente de les coller avec + et d'afficher le tout. Observe l'erreur.

💡 Indice 1

Écris simplement print(message + age). L'objectif est de voir ce qui se passe quand on mélange texte et nombre.

💡 Indice 2

Python ne sait pas s'il doit additionner ou concaténer : il refuse et lève une TypeError.

✅ Solution
message = "Mon âge est "
age = 30

# La ligne ci-dessous déclenche une TypeError :
# print(message + age)

Pourquoi cette erreur :

  • Message exact : TypeError: can only concatenate str (not "int") to str.
  • Python est fortement typé : il ne mélange pas les types implicitement. On ne « colle » pas un entier à une chaîne. La solution est la conversion manuelle — le casting, à l'exercice suivant.
Attention — deux pièges qui coûtent des heures (le vécu terrain)
  • input() renvoie TOUJOURS du texte, même si l'utilisateur tape 30. Faire input() + 1 plante donc avec une TypeError. Il faut convertir : int(input(...)).
  • La virgule décimale française casse float() : float("3,14") lève une ValueError car Python attend un point. C'est LE bug des fichiers CSV français — on remplace d'abord la virgule : float("3,14".replace(",", ".")).

Exercice 5 — Le casting à la rescousse

Reprends l'exercice 4 et corrige l'erreur en convertissant age en chaîne de caractères avant de le concaténer.

💡 Indice 1

Pour convertir une valeur en texte, on utilise la fonction str().

💡 Indice 2

La syntaxe : message + str(age). La f-string, elle, fait la conversion toute seule.

✅ Solution
message = "Mon âge est "
age = 30

# Solution 1 — casting explicite avec str() :
print(message + str(age))

# Solution 2 — la f-string convertit automatiquement :
print(f"{message}{age}")

# Le casting marche dans les deux sens :
altitude = int("250") + 5      # texte → entier : 255
prix = float("19.90")            # texte → décimal : 19.9

Comment ça marche :

  • Casting = convertir une valeur d'un type à un autre. str(age) transforme l'entier 30 en texte "30" ; l'opération message + str(age) colle alors deux chaînes : c'est valide.
  • Dans l'autre sens, int("250") et float("19.90") transforment du texte en nombre — indispensable pour calculer à partir de données lues dans un fichier.

Exercice 6 — Dialogue avec le grimpeur

Demande son altitude actuelle à l'utilisateur avec input(), ajoute 150 m de progression, et affiche la nouvelle altitude. Attention au type !

💡 Indice 1

Tu peux guider l'utilisateur : input("À quelle altitude es-tu ? ").

💡 Indice 2

Comme input() renvoie du texte, convertis-le en entier avec int() avant de calculer.

✅ Solution
altitude_str = input("À quelle altitude es-tu (en mètres) ? ")
altitude = int(altitude_str)        # input() renvoie du texte : on convertit
nouvelle_altitude = altitude + 150

print(f"Après 150 m de plus, tu seras à {nouvelle_altitude} mètres.")

Comment ça marche :

  • input("...") affiche le message, attend la saisie, puis renvoie ce que l'utilisateur a tapé — toujours sous forme de texte (str).
  • int(altitude_str) convertit ce texte en entier pour pouvoir l'additionner.
  • Si l'utilisateur tape « beaucoup » au lieu d'un nombre, int() lèvera une ValueError. On apprendra à gérer ce cas proprement au Camp 7, avec try / except.
Astuce de pro

Pour afficher un calcul à virgule sans une ribambelle de décimales, arrondis : round(moyenne, 1) → une décimale. Et garde un réflexe « data » : convertis tes valeurs explicitement (int(), float(), str()) plutôt que de compter sur la magie des f-strings. Quand tu nettoieras un fichier de milliers de lignes, savoir exactement le type de chaque colonne t'évitera des résultats faux et silencieux — le pire genre de bug.

Exercice 7 — Répartir les cordées

Il y a 27 grimpeurs, à répartir en cordées de 4. Combien de cordées complètes ? Combien de grimpeurs restent seuls ? Affiche les deux.

💡 Indice 1

La division entière // donne le quotient sans virgule.

💡 Indice 2

L'opérateur modulo % donne le reste de la division.

✅ Solution
grimpeurs = 27
taille_cordee = 4

cordees_completes = grimpeurs // taille_cordee   # division entière : 6
restants = grimpeurs % taille_cordee             # reste : 3

print(f"{cordees_completes} cordées complètes, {restants} grimpeurs restants.")
  • / donne un décimal (27 / 4 = 6.75), // donne l'entier (6), % donne le reste (3).
  • Le modulo % est un couteau suisse : tester si un nombre est pair (n % 2 == 0), répartir, faire des cycles… tu le reverras souvent.

Exercice 8 — Puissance et priorité

Affiche 2 puissance 10. Puis affiche 2 + 3 * 4 et (2 + 3) * 4, et observe la différence.

💡 Indice 1

L'opérateur puissance est ** : 2 ** 10.

💡 Indice 2

Comme en maths, la multiplication passe avant l'addition. Les parenthèses changent l'ordre.

✅ Solution
print(2 ** 10)          # 1024

print(2 + 3 * 4)        # 14 : la multiplication d'abord
print((2 + 3) * 4)      # 20 : les parenthèses changent l'ordre
  • ** élève à la puissance. La priorité des opérateurs suit les maths : puissance, puis * /, puis + -.
  • Dans le doute, mets des parenthèses : c'est plus sûr et surtout plus lisible pour celui qui te relit.

Exercice 9 — Nettoyer une saisie

On reçoit nom = " aLeX " (espaces parasites, casse incohérente). Obtiens Alex bien propre, puis affiche le nombre de caractères du résultat.

💡 Indice 1

.strip() retire les espaces au début et à la fin. .capitalize() met la première lettre en majuscule et le reste en minuscule.

💡 Indice 2

On peut enchaîner les méthodes : nom.strip().capitalize(). len() compte les caractères.

✅ Solution
nom = "  aLeX  "

propre = nom.strip().capitalize()   # enlève les espaces, puis met en forme
print(propre)        # Alex
print(len(propre))   # 4

# Autres méthodes très utiles :
print(nom.upper())    # "  ALEX  "
print(nom.replace(" ", ""))  # "aLeX" (supprime tous les espaces)
  • Les méthodes de chaîne s'appellent avec un point : texte.methode(). Elles renvoient une nouvelle chaîne (le texte original ne change pas).
  • .strip() est LE réflexe du data analyst : les espaces invisibles en début/fin de cellule sont la cause n°1 de doublons « fantômes » dans un fichier.

Exercice 10 — Comparer des valeurs

Avec altitude = 8200, affiche le résultat (vrai ou faux) de : altitude supérieure à 8000, altitude égale à 8848, et altitude différente de 0.

💡 Indice 1

Les comparaisons renvoient un booléen (True ou False) : >, <, >=, <=.

💡 Indice 2

Attention : l'égalité se teste avec un double égal == (le simple = sert à affecter une variable). Le « différent de » s'écrit !=.

✅ Solution
altitude = 8200

print(altitude > 8000)    # True
print(altitude == 8848)   # False
print(altitude != 0)      # True
  • Une comparaison répond toujours par True ou False. C'est exactement ce dont les conditions (Camp 4) ont besoin.
  • = affecte une valeur, == teste l'égalité : confondre les deux est l'erreur la plus fréquente du débutant.

Exercice 11 — Mettre en forme les nombres

Avec prix = 19.9, ratio = 0.732 et grand = 1250000, affiche respectivement : 19.90 €, 73% et 1,250,000.

💡 Indice 1

Dans une f-string, on précise un format après deux-points : f"{prix:.2f}".

💡 Indice 2

.2f = 2 décimales, .0% = pourcentage sans décimale, , = séparateur de milliers.

✅ Solution
prix = 19.9
ratio = 0.732
grand = 1250000

print(f"{prix:.2f} €")     # 19.90 €   -> 2 décimales
print(f"{ratio:.0%}")      # 73%       -> pourcentage
print(f"{grand:,}")        # 1,250,000 -> séparateur de milliers
  • Après les deux-points, la f-string accepte un mini-langage de formatage très puissant.
  • Indispensable pour des tableaux de bord et des rapports propres : des prix alignés, des pourcentages lisibles, de grands nombres aérés.
Tu sais maintenant…
  • Faire des calculs avec + - * / // % ** et gérer la priorité des opérateurs ;
  • Reconnaître les types de base — int, float, str, bool — et les vérifier avec type() ;
  • Concaténer et nettoyer du texte (+, .strip(), .upper(), .replace(), len()) ;
  • Repérer et corriger une TypeError grâce au casting (str(), int(), float()) ;
  • Comparer des valeurs (== != > < >= <=) et obtenir des booléens ;
  • Lire une saisie avec input(), mettre en forme les nombres, et éviter le piège de la virgule décimale.

Tu jongles déjà avec les types de données. La montagne semble moins intimidante.

Exercice 1 — Créer sa première liste

Crée une liste equipement contenant "corde", "piolet", "casque", puis affiche-la.

Exercice 11 — Décision multi-critères

Avec temperature = 5, vent_fort = False, presence_neige = True, détermine l'état de l'ascension :

💡 Indice 2

Concis, mais à réserver aux cas simples — au-delà, un vrai if/else reste plus lisible.

Boucles & itérations

for · while · range() · break / continue · compréhensions — automatiser

✓ Camp franchi

Pourquoi répéter 100 fois la même chose à la main ? Les boucles automatisent les tâches répétitives et font gagner un temps précieux — l'outil préféré du data analyst.

Exercice 1 — Parcourir l'équipement

Avec equipement = ["corde", "piolet", "casque", "baudrier"], affiche chaque élément sur une ligne avec une boucle for.

💡 Indice 1

La syntaxe : for element in ma_liste: puis un bloc indenté.

💡 Indice 2

La variable de boucle prend successivement la valeur de chaque élément.

✅ Solution
equipement = ["corde", "piolet", "casque", "baudrier"]

for item in equipement:
    print(item)
  • for item in equipement: se lit « pour chaque item dans la liste ».
  • Le bloc indenté s'exécute une fois par élément.

Exercice 2 — Compter de 1 à 10

Affiche les nombres de 1 à 10 avec une boucle for et la fonction range().

💡 Indice 1

range(debut, fin) génère des nombres de debut (inclus) à fin (exclu).

💡 Indice 2

Pour aller de 1 à 10, utilise range(1, 11).

✅ Solution
# La fin (11) est exclue : la boucle s'arrête à 10.
for nombre in range(1, 11):
    print(nombre)
  • range() génère une séquence sans qu'on ait à écrire la liste à la main.
  • Piège classique : la borne de fin est exclue. range(1, 11) s'arrête à 10.
Le savais-tu ? — range() est paresseux

range(1, 1000000) ne crée pas un million de nombres en mémoire ! C'est un objet « paresseux » (lazy) qui fabrique chaque valeur à la demande, une par une, pendant la boucle. Résultat : la mémoire utilisée reste minuscule, même pour des milliards d'itérations. Cette idée de « produire au fur et à mesure » plutôt que « tout charger d'un coup » est au cœur de la performance en data — tu la retrouveras avec les générateurs et le traitement de gros fichiers par morceaux (chunks).

Exercice 3 — La somme des paliers

Avec paliers = [120, 250, 180, 300], calcule l'altitude totale gagnée et affiche-la.

💡 Indice 1

Crée un accumulateur total = 0 avant la boucle.

💡 Indice 2

À chaque tour, ajoute la valeur courante avec total += palier.

✅ Solution
paliers = [120, 250, 180, 300]

    total += palier   # raccourci de : total = total + palier

print(f"Altitude totale gagnée : {total} mètres.")

# Astuce : sum() fait la somme directement
print(sum(paliers))   # 850
  • Le schéma « accumulateur » (init à 0, on ajoute dans la boucle) sert à faire des sommes, des moyennes, des comptages.
  • Le print() est placé après la boucle pour n'afficher que le total final.

Affiche les nombres pairs de 0 à 10 (0, 2, 4, 6, 8, 10) en utilisant le pas de range().

Exercice 5 — Trouver et s'arrêter (break)

Dans equipement = ["sac", "piolet", "gants", "casque", "lampe"], vérifie si le "casque" est présent : si oui, affiche Équipement de sécurité vérifié ! et arrête de chercher.

💡 Indice 1

Dans la boucle, teste chaque élément avec un if.

💡 Indice 2

break interrompt la boucle immédiatement.

✅ Solution
equipement = ["sac", "piolet", "gants", "casque", "lampe"]

    if item == "casque":
        print("Équipement de sécurité vérifié !")

Exercice 6 — Ignorer certains objets (continue)

Avec objets = ["piolet", "livre", "casque", "téléphone"], affiche tous les objets sauf "livre" et "téléphone".

💡 Indice 1

continue saute le reste du tour et passe directement à l'élément suivant.

💡 Indice 2

On peut tester l'appartenance à une liste à ignorer avec not in.

✅ Solution
objets = ["piolet", "livre", "casque", "téléphone"]

for objet in objets:
    if objet in ["livre", "téléphone"]:
        continue   # on ignore et on passe au suivant
    print(objet)
  • continue n'arrête pas la boucle (contrairement à break) : il saute juste le tour courant.

Exercice 7 — Le compte à rebours (while)

Avec une boucle while, fais un compte à rebours de 5 à 1, puis affiche Décollage !

💡 Indice 1

La boucle while condition: tourne tant que la condition est vraie.

💡 Indice 2

Décrémente le compteur (compteur -= 1) à chaque tour, sinon la boucle ne s'arrête jamais.

✅ Solution
compteur = 5

while compteur > 0:
    print(compteur)
    compteur -= 1   # indispensable !

print("Décollage !")
  • La boucle for sert quand on connaît le nombre de tours ; la boucle while sert quand on s'arrête sur une condition.
  • Sans la décrémentation, compteur resterait à 5 → boucle infinie.
Attention — deux pièges des boucles
  • La boucle infinie : dans un while, si la condition ne devient jamais fausse (oubli d'incrément, mauvais test), le programme tourne sans fin. Dans un notebook, le bouton « Stop » (carré) interrompt l'exécution.
  • Modifier une liste pendant qu'on la parcourt (ajouter/supprimer des éléments dans le for) provoque des comportements imprévisibles. La bonne pratique : construire une nouvelle liste plutôt que modifier celle qu'on parcourt.

Exercice 8 — Le mot de passe (while True + break)

Demande un mot de passe en boucle. Tant que ce n'est pas "Everest", redemande. Quand c'est bon, affiche Accès autorisé.

💡 Indice 1

💡 Indice 2

Demande la saisie dans la boucle, et sors dès que le mot de passe est correct.

while True:

Exercice 9 — Numéroter les étapes (enumerate)

Avec etapes = ["Camp 1", "Camp 2", "Sommet"], affiche : Étape 1 : Camp 1, Étape 2 : Camp 2, Étape 3 : Sommet.

💡 Indice 1

enumerate(liste) fournit à chaque tour un couple (index, valeur).

💡 Indice 2

Ajoute start=1 pour que la numérotation commence à 1 plutôt qu'à 0.

Astuce de pro — la trousse à itérer

Trois outils que les pros utilisent sans cesse : enumerate(liste) pour l'index + la valeur ; zip(liste_a, liste_b) pour parcourir deux listes en parallèle (par ex. des noms et des altitudes) ; et la compréhension de liste pour transformer/filtrer en une ligne. Ajoute sum(), min(), max(), et les redoutables any() / all() (« au moins un est vrai » / « tous sont vrais ») : avec eux, beaucoup de boucles disparaissent et le code devient bien plus court à lire.

Exercice 10 — La grille de coordonnées (boucles imbriquées)

Affiche toutes les paires (x, y) d'une grille 3×3, où x et y vont de 1 à 3, une ligne de la grille par ligne d'affichage.

💡 Indice 1

Place une boucle for à l'intérieur d'une autre. L'extérieure gère x, l'intérieure gère y.

💡 Indice 2

print(..., end=" ") reste sur la même ligne ; un print() vide passe à la ligne suivante.

Exercice 11 — Parcourir un dictionnaire

Avec grimpeur = {"nom": "Messner", "pays": "Italie", "altitude_max": 8848}, affiche chaque paire sous la forme clé : valeur.

Exercice 12 — La compréhension de liste

Crée la liste des carrés des nombres de 1 à 5, d'abord avec une boucle classique, puis en une seule ligne. Bonus : la liste des nombres pairs de 1 à 10.

La forme : [expression for x in sequence].

💡 Indice 2

Fonctions

def · paramètres · return · valeurs par défaut · docstrings · portée — créer ses outils

✓ Camp franchi

Le but de tout programmeur : écrire du code réutilisable. Avec les fonctions, on crée ses propres commandes pour résoudre des problèmes précis — et on arrête de se répéter.

Exercice 1 — Ta première fonction

Crée une fonction saluer() qui affiche Bienvenue sur le Mont Python !, puis appelle-la.

On définit une fonction avec def nom(): suivi d'un bloc indenté.

Définir ne suffit pas : il faut appeler la fonction avec saluer().

✅ Solution
def saluer():

def saluer(prenom):

saluer("Alex")
saluer("Marie")
  • Le paramètre est le nom dans la définition ; l'argument est la valeur passée à l'appel.

Exercice 3 — Renvoyer une valeur (return)

Crée carre(n) qui renvoie le carré de n. Stocke carre(5) dans une variable et affiche-la, puis affiche directement carre(8).

💡 Indice 1

return renvoie une valeur à celui qui a appelé la fonction.

💡 Indice 2

Différence clé : print affiche, return renvoie une valeur réutilisable.

✅ Solution
def carre(n):

print(carre(8))   # 64
  • Une fonction qui return produit un résultat qu'on peut stocker, réafficher, ou réinjecter dans un autre calcul.
  • Avec print seul, la valeur s'affiche mais est perdue : impossible de la réutiliser.
Le savais-tu ? — En Python, les fonctions sont des objets

Une fonction peut être rangée dans une variable, mise dans une liste, ou passée en argument à une autre fonction — au même titre qu'un nombre. On dit qu'elles sont des « citoyennes de première classe ». Cela rend possible des outils très puissants comme sorted(equipe, key=...), où l'on passe une fonction pour dire comment trier. Pour les cas tout simples, Python propose même la fonction-éclair lambda : doubler = lambda x: x * 2. À découvrir plus tard, mais bon à savoir : c'est la porte d'entrée vers le style « fonctionnel ».

Crée aire_rectangle(largeur, hauteur) qui renvoie l'aire. Affiche le résultat pour 4 × 5.

💡 Indice 1

On sépare les paramètres par des virgules : def aire_rectangle(largeur, hauteur):.

💡 Indice 2

L'ordre des arguments compte : le premier va dans largeur, le second dans hauteur.

✅ Solution
def aire_rectangle(largeur, hauteur):
    return largeur * hauteur

print(aire_rectangle(4, 5))   # 20
  • Les arguments sont associés aux paramètres par position, dans l'ordre.
  • On peut aussi nommer les arguments à l'appel : aire_rectangle(hauteur=5, largeur=4).

Exercice 5 — Un argument par défaut

Crée saluer(prenom, message="Bon courage !") qui affiche {prenom} : {message}. Appelle-la une fois sans message, une fois avec un message personnalisé.

💡 Indice 1

Une valeur par défaut se déclare avec = dans la définition : message="...".

💡 Indice 2

Les paramètres avec défaut doivent venir après ceux sans défaut.

✅ Solution
def saluer(prenom, message="Bon courage !"):

Attention — trois subtilités des fonctions
  • Le défaut mutable : ne mets jamais une liste ou un dictionnaire comme valeur par défaut (def f(items=[]):). Cette liste est partagée entre tous les appels → bugs sournois. Utilise None puis crée la liste à l'intérieur.
  • return None implicite : une fonction sans return renvoie None. Si tu écris x = ma_fonction() alors qu'elle ne fait qu'un print, x vaudra None.
  • La portée : une fonction peut lire une variable globale, mais pour transmettre un résultat à l'extérieur, le bon réflexe est de le renvoyer avec return (pas de bricolage avec global).

Crée filtrer_hautes(altitudes, seuil) qui renvoie une nouvelle liste avec seulement les altitudes au-dessus du seuil. Teste avec [2400, 8848, 3600, 8201] et un seuil de 8000.

Exercice 12 — La portée des variables

Exercice 1 — Gérer une erreur (try / except)

Demande un âge avec input() et affiche l'âge de l'an prochain. Si l'utilisateur ne tape pas un nombre, affiche Ce n'est pas un nombre valide. au lieu de planter.

💡 Indice 1

Place le code risqué dans un bloc try:, et la gestion de l'erreur dans except:.

💡 Indice 2

Convertir un texte non numérique en entier déclenche une ValueError.

✅ Solution
try:
    age = int(input("Ton âge ? "))
    print(f"L'an prochain, tu auras {age + 1} ans.")
except ValueError:
    print("Ce n'est pas un nombre valide.")
  • Sans try/except, une saisie invalide ferait planter tout le programme avec une erreur rouge.
  • Avec, le programme réagit proprement et continue. C'est la base d'un code robuste.

Exercice 2 — Attraper la bonne erreur

Crée diviser(a, b) qui renvoie a / b, mais renvoie Division par zéro impossible. si b vaut 0. Teste avec (10, 2) et (10, 0).

💡 Indice 1

Diviser par zéro déclenche une ZeroDivisionError : c'est elle qu'on attrape.

💡 Indice 2

On précise le type après except pour ne capturer que cette erreur-là.

✅ Solution
def diviser(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        return "Division par zéro impossible."

print(diviser(10, 2))   # 5.0
print(diviser(10, 0))   # Division par zéro impossible.
  • Chaque problème a son type d'exception : ValueError, ZeroDivisionError, KeyError, TypeError, FileNotFoundError…
  • Cibler le bon type évite de masquer d'autres bugs par accident.
Attention — le except « nu », à bannir

Écrire except: tout seul (ou except Exception: sans réfléchir) attrape toutes les erreurs, y compris celles que tu n'avais pas prévues — et masque ainsi de vrais bugs. C'est l'un des pires réflexes en Python. Attrape toujours le type le plus précis possible. Si tu dois vraiment capturer largement, garde au moins le détail : except Exception as e: puis affiche e pour savoir ce qui s'est passé. Un programme qui « avale » ses erreurs en silence est bien plus dangereux qu'un programme qui plante franchement.

Exercice 3 — La structure complète (else / finally)

Demande une altitude. En cas de saisie invalide, affiche un message d'erreur. Si tout va bien, affiche l'altitude enregistrée. Dans tous les cas, affiche Vérification terminée.

💡 Indice 1

else s'exécute uniquement si aucune erreur n'a eu lieu dans le try.

💡 Indice 2

finally s'exécute toujours, qu'il y ait eu une erreur ou non.

✅ Solution
try:
    altitude = int(input("Altitude (m) ? "))
except ValueError:
    print("Saisie invalide.")
else:
    print(f"Altitude enregistrée : {altitude} m.")   # si AUCUNE erreur
finally:
    print("Vérification terminée.")                  # TOUJOURS
  • else sépare clairement « ce qui peut échouer » (try) de « ce qui suit en cas de succès ».
  • finally sert au nettoyage garanti : fermer une connexion, un fichier… même si une erreur survient.

Exercice 4 — Lever sa propre exception (raise)

Crée enregistrer_altitude(metres) qui renvoie l'altitude, mais déclenche une ValueError avec un message clair si l'altitude est négative.

💡 Indice 1

raise ValueError("message") déclenche volontairement une exception.

💡 Indice 2

On valide d'abord, puis on lève l'erreur si la donnée est invalide.

✅ Solution
def enregistrer_altitude(metres):
    if metres < 0:
        raise ValueError("L'altitude ne peut pas être négative.")
    return metres

print(enregistrer_altitude(2400))   # 2400
# enregistrer_altitude(-50)          -> ValueError: L'altitude ne peut pas être négative.
  • raise sert à refuser une donnée invalide tôt, avec un message explicite (« échouer vite »).
  • Mieux vaut une erreur claire au bon endroit qu'un résultat faux qui se propage silencieusement.

Avec les modules math et random, affiche la racine carrée de 144, la valeur de pi, et un entier aléatoire entre 1 et 6.

On charge un module avec import nom_module en haut du fichier.

Exercice 6 — Alias et import ciblé

Importe statistics sous l'alias stats, et importe pi et sqrt directement depuis math. Calcule la moyenne de [12, 15, 9, 18] et la racine de 81.

💡 Indice 1

import statistics as stats crée un alias plus court.

💡 Indice 2

from math import pi, sqrt rend ces noms accessibles directement, sans préfixe.

✅ Solution
import statistics as stats      # alias
from math import pi, sqrt        # import ciblé

print(sqrt(81))            # 9.0  (pas besoin de math.)

Exercice 7 — Créer son propre module

Comprends comment ranger des fonctions réutilisables dans un fichier outils_montagne.py, puis les importer ailleurs.

On l'importe par son nom de fichier, sans l'extension .py.

Exercice 8 — Lire et écrire un fichier (with open)

Écris deux lignes dans journal.txt, puis relis tout le fichier et affiche son contenu.

Exercice 10 — Éliminer les « nombres magiques »

Le nombre 8848 (altitude de l'Everest) revient sans explication. Remplace-le par une constante nommée, puis écris reste_a_gravir(altitude).

Une constante se nomme en MAJUSCULES et se déclare en haut du fichier.

💡 Indice 2

La fonction utilise la constante au lieu d'écrire 8848 en dur.

ALTITUDE_SOMMET = 8848   # constante : un seul endroit à modifier

print(reste_a_gravir(6000))   # 2848

Exercice 11 — La fonction exemplaire (typage + docstring)

Écris moyenne(nombres) qui calcule la moyenne d'une liste, avec des annotations de type et une docstring complète.

Les annotations : def moyenne(nombres: list) -> float:.

💡 Indice 2

def moyenne(nombres: list) -> float:
    """Calcule la moyenne d'une liste de nombres.

    Argument :
        nombres (list) : liste de valeurs numériques.
    Retour :
        float : la moyenne arithmétique.

    return sum(nombres) / len(nombres)

print(moyenne([10, 20, 30]))   # 20.0
